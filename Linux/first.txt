# 1 Linux File System
	ext:
		리눅스 운영체제를 만들기 위한 첫 파일시스템
		Extended File System
		파일시스템 최대크기 2GB //한 파티션당 최대 크기
		파일 패스 255byte
		사용할수록 시스템이 조각화됨(단편화)
## ext2:
		파일명 최대 256byte
		단일파일 최대 2GB
		디스크 최대 4TB사용가능
		e2fsck으로 비정상종료시 시스템 검사가능
##	ext3:
		리눅스 커널2.4, 레드햇 버전 7.2부터 적용
		ext2에 저널링기능 옮긴것 (강제종료시 손상될 위험성 낮아짐)
		작업수행속도 빨라짐
		16TB 파일시스템 인식, 자식 디렉토리 최대 31998개 가능
		삭제한 파일은 복구가 잘 안됨
##	ext4:
		1EB까지 디스크, 16TB까지의 파일크기 지원
		이전의 block mapping방식 대신 extends방식으로 큰파일 처리와 단편화 개선
		하위버전은 ext4로 호환되지만 ext4를 ext3에는 못물린다.
		저널링 기능에서 저널 체크섬 기능 추가, 파일 손상확률 더 낮아짐
		자식 디렉토리 최대 64000개 가능
		온라인 조각모음 지원
		사용하지 않는 부분은 스킵하여 시스템 검사를 빠르게 할 수 있다.
		타임스탬프가 나노초단위로 정교해졌다.
		디스크 선행할당 할때 0으로 초기화 안해도 가능
		확장자가 생겨났다.

# 2 Disk 설치 과정
	fdisk:
		fdisk -l	//장치 파일명 확인
		n	//파티션 생성
			파티션 종류 : e, p
			파티션 번호 : 1~4
			실린더 번호 : 1~1024
			파티션 크기 : M, K, B 지정가능
		p	//파티션 만들어졌는지 확인, id 코드도 나옴
		w	//저장후 종료
		q	//저장하지 않고 종료
		d	//파티션 삭제
		l	//알려진 파일시스템 코드 보통 83 linux 시스템
	mkfs: 포맷하는것임
		mkfs -t [ext2, ext3, ext4] [옵션] 장치명
		-v	//결과를 상세하게 출력
		-c	//배드블록 체크 후 파일시스템 구축
		ex) mkfs -t ext4 /dev/sda1
	mount: 
		mount -t [ext2,ext3,ext4] [dev] [dir]
		ex) mount /dev/sda3 /multi_zoa
		umount [dir]	//마운트 해제
		/etc/fstab에 UUID와 디렉토리를 지정해주면 자동마운트 된다.
		꿀팁) 터미널에 blkid치면 장치 UUID 쫙뜸

# 3 Link

	Symbolic Link:
		바로가기, 원본파일의 포인터라고 생각하면 됨
	inode:

# 4 Shell
	쉘의 특징:
		사용자가 입력하는 명령을 해석하여 실행해주는 프로그램
		다른 OS에서도 Shell은 존재
		쉘에 대한 이해는 운영체제에 대한 이해!
	쉘 종류: (/bin/아래 존재)
		sh	//대부분 UNIX, LINUX 쉘
		bash	//Linux기본쉘, sh랑 호환
		ksh	//벨연구소에서 개발, 부동소수점 계산 쌉가능
		csh	//C언어랑 비슷한 스타일로 쉘스크립트 작성가능
		tcsh	//csh에 기능을 추가한것
	쉘과 커널:
		쉘은 커널과 운영체제 사이에서 사용자의 입력을 대신 수행해준다.
		exec()함수로 입력된 값 실행, 인터럽트 제어권한을 프로그램에게 넘겨주고 대기

# 5 Shell Script
	쉘스크립트 사용 장점:
		1. 반복작업에 대한 자동화가 가능하다. 스케줄 설정 등 주기적으로 하는일에 활용가능
		2. 기존의 명령어를 개인화한다. 명령어 실행결과를 원하는대로 조작가능 (재지정)
		3. 유지보수가 쉽다. 별도의 컴파일러가 필요없다.
	쉘스크립트 사용 단점:
		1. 실행되는 명령어 자식 프로세스의 필요에 따라 느려질 수도 있다.
		2. 복잡한 쉘 스크립트는 호환성이 구데기일 수도 있다.
		3. 제한된 틀 안에서 사용가능하기때문에 고품질의 코드와 확장기록 힘들음.
	쉘 변수 \붙이면 어캐되는지:
		'$변수명' 자체가 문자열이 되어서 나온다.
	echo 'asdf':
		싱글쿼터 안에는 뭐가들어가던 문자열 처리
	환경변수:
		미리 설정된 명령이나 변수값이라고 생각하면 편할듯
		대표적인 사전설정된 환경변수:
			$DISPLAY		//x프로그램 서버 위치정보(뭔개소리지)
			$HOME		//사용자 홈디렉토리
			$HOSTNAME	//호스트네임
			$PATH		//검색 경로
			$PSI		//프롬프트 설정값
			$PWD		//pwd명령이랑 같음
			$SHELL		//지금 쓰는 쉘 경로
			$TERM		//터미널 종류
			$MANPATH	//man 페이지 경로
			$LS_COLORS	//ls 명령 색깔지정값
			$MAIL		//메일이 저장된 파일경로
			$MAILCHECK	//메일 새로고침하는 초단위 값
		#echo $변수이름 or #printenv 변수명
		printenv하면 모든 환경변수 확인

			vsftpd.service	//FTP
			named		//DNS

		chkconfig --list
		chkconfig httpd on
		chkconfig httpd off
